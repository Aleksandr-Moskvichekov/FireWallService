# Инструкция по использованию FireWallService и FirewallWiewApp

## Быстрый старт

### Шаг 1: Запустите FireWallService
1. Откройте PowerShell **от имени администратора**
2. Перейдите в папку `FireWallService\bin\Debug\net10.0` (или Release)
3. Запустите команду:
   ```powershell
   .\FireWallService.exe
   ```
4. Вы должны увидеть сообщение:
   ```
   ╔════════════════════════════════════════════════════════╗
   ║   FireWallService Started                              ║
   ║   Named Pipe: FireWallServicePipe                      ║
   ║   Waiting for GUI connections...                       ║
   ║════════════════════════════════════════════════════════╝
   ```

### Шаг 2: Запустите FirewallWiewApp
1. В другом PowerShell окне (также от администратора)
2. Перейдите в папку `FirewallWiewApp\bin\Debug\net10.0-windows`
3. Запустите:
   ```powershell
   .\FirewallWiewApp.exe
   ```

### Шаг 3: Проверьте подключение
1. Нажмите кнопку **"Диагностика"** в правом верхнем углу главного окна
2. Нажмите **"Test Connection"**
3. Если подключение успешно, вы увидите:
   ```
   [HH:mm:ss] ✓ Successfully connected to FireWallService!
   [HH:mm:ss] ✓ Got status: Running=True, Rules=N
   ```

## Функции приложения

### Управление правилами (вкладка "Правила")
- Просмотр всех правил файервола
- Добавление новых правил
- Удаление существующих правил
- Редактирование правил

### Просмотр логов (вкладка "Логи")
- Просмотр всех событий файервола
- Фильтрация по типу
- Очистка логов

### Статистика (вкладка "Статистика")
- Общая информация о работе сервиса
- Количество обработанных пакетов
- Статистика блокировок

### Захват пакетов (вкладка "Пакеты")
- Просмотр недавно обработанных пакетов
- Информация о каждом пакете
- Запись пакетов в pcap файл

### Параметры (вкладка "Параметры")
- Настройка поведения файервола
- Включение/отключение захвата пакетов
- Выбор режима работы

## Диагностика проблем

### Если подключение не работает

#### 1. Проверьте права доступа
- **Оба приложения должны работать от имени администратора**
- Щелкните правой кнопкой → "Запустить от имени администратора"

#### 2. Проверьте, запущен ли сервис
В PowerShell:
```powershell
Get-Process | Where-Object {$_.Name -like "*FireWall*"}
```

Вы должны увидеть `FireWallService` в списке

#### 3. Проверьте логи
- Логи сервиса: `firewall.log` в папке запуска сервиса
- Логи приложения: откройте Diagnostics → "Auto-Test" и смотрите логи

#### 4. Отключите Firewall Windows
Если Windows Firewall блокирует именованные каналы:
```powershell
# Отключить (временно)
Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled $False

# Включить обратно
Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled $True
```

### Типичные ошибки

| Ошибка | Причина | Решение |
|--------|---------|---------|
| Connection timeout | Сервис не запущен | Запустите FireWallService.exe |
| Access is denied | Недостаточно прав | Запустите от администратора |
| Empty response | Сервис не ответил | Проверьте firewall.log |
| Connection reset | Сервис упал | Перезагрузите сервис |

## Команды консоли FireWallService

При запуске FireWallService в консоли вводите команды:

```
help              - показать эту справку
status            - показать статус сервиса
rules             - показать все правила
packets           - показать последние пакеты
stats             - показать статистику
addrule           - добавить новое правило
removerule        - удалить правило
clear-packets     - очистить историю пакетов
clear-logs        - очистить логи
start             - начать захват пакетов
stop              - остановить захват
exit              - выход
```

## Требования

- Windows 7 или выше
- .NET 10.0 (установлен на компьютер)
- Права администратора
- Установленные WinDivert драйверы

## Лицензия и лицензионное соглашение

Это приложение использует WinDivert для перехвата пакетов.
Убедитесь, что вы ознакомились с лицензией WinDivert.

## Поддержка

Если у вас возникли проблемы:
1. Проверьте CONNECTION_DIAGNOSTICS.md
2. Включите Advanced Logging в settings.json
3. Отправьте firewall.log для анализа
