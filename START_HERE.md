# 🚀 НАЧАТЬ ЗДЕСЬ - Инструкция запуска

## Если у вас проблема с подключением к сервису, выполните следующее:

### Шаг 1: Откройте PowerShell от администратора

Нажмите `Win + X` и выберите "Windows PowerShell (Admin)" или:

```powershell
Start-Process powershell -Verb RunAs
```

### Шаг 2: Запустите FireWallService

Скопируйте и выполните в PowerShell:

```powershell
cd "D:\Документы\MVS Programs\FireWallService\FireWallService\bin\Debug\net10.0"
.\FireWallService.exe
```

**Вы должны увидеть это сообщение:**
```
╔════════════════════════════════════════════════════════╗
║   FireWallService Started                              ║
║   Named Pipe: FireWallServicePipe                      ║
║   Waiting for GUI connections...                       ║
╚════════════════════════════════════════════════════════╝
```

⚠️ **ВАЖНО:** Оставьте это окно открытым!

### Шаг 3: Откройте второе окно PowerShell

Нажмите `Win + X` и выберите "Windows PowerShell (Admin)" еще раз.

### Шаг 4: Запустите приложение

В новом окне PowerShell выполните:

```powershell
cd "D:\Документы\MVS Programs\FireWallService\FirewallWiewApp\bin\Debug\net10.0-windows"
.\FirewallWiewApp.exe
```

Должно открыться окно приложения с вкладками "Правила", "Логи" и т.д.

### Шаг 5: Проверьте подключение

1. В приложении найдите кнопку **"Диагностика"** в правом верхнем углу
2. Нажмите **"Диагностика"**
3. В открывшемся окне нажмите **"Test Connection"**

**Если вы видите:**
```
✓ SUCCESS: Service is running
```

Поздравляем! 🎉 Подключение работает!

**Если вы видите ошибку**, смотрите раздел "Решение типичных ошибок" ниже.

## 🔴 Решение типичных ошибок

### Ошибка: "Connection timeout"

**Это означает:** Сервис не запущен или не слушает канал

**Как исправить:**
1. Убедитесь, что первое окно PowerShell (с сервисом) все еще открыто
2. Должно быть сообщение "Waiting for GUI connections..."
3. Если окна нет - перезапустите сервис (шаг 2)

### Ошибка: "Access is denied"

**Это означает:** Недостаточно прав администратора

**Как исправить:**
1. Закройте оба приложения
2. Откройте PowerShell от администратора (Нажмите правой кнопкой на PowerShell → "Запустить от имени администратора")
3. Повторите шаги 2 и 4

### Ошибка: "Empty response from service"

**Это означает:** Сервис не отвечает правильно

**Как исправить:**
1. Посмотрите в консоли сервиса (первое окно PowerShell)
2. Должны быть сообщения типа "GUI client connected"
3. Если их нет - перезагрузите приложение (нажмите кнопку "Диагностика" снова)

## 🆘 Если возникает проблема после запуска

### Приложение не открывается
```powershell
# Проверьте, установлен ли .NET 10
dotnet --list-runtimes
```
Должно быть что-то типа: `Microsoft.WindowsDesktop.App 10.0.x`

Если нет - установите .NET 10 Runtime с официального сайта Microsoft.

### Сервис не запускается
```powershell
# Попробуйте пересобрать проект
cd "D:\Документы\MVS Programs\FireWallService"
dotnet clean
dotnet build
```

### Все еще не работает?
1. Перезагрузитесь
2. Запустите оба приложения заново от администратора
3. Нажмите "Clear Logs" в Diagnostics Window
4. Запустите "Auto-Test" и смотрите результаты

## 📋 Чеклист перед запуском

Убедитесь что:

- [ ] Windows PowerShell открыт от администратора
- [ ] Путь `D:\Документы\MVS Programs\FireWallService` существует
- [ ] .NET 10 Runtime установлен (проверить: `dotnet --list-runtimes`)
- [ ] Windows Firewall не блокирует приложения (или отключен)
- [ ] Нет других экземпляров приложений запущено
