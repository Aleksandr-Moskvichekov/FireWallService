# Улучшения для диагностики проблемы подключения IPC

## Что было сделано

### 1. 🔍 Улучшенное логирование IpcClient

**Файл:** `FirewallWiewApp/Services/IpcClient.cs`

#### Добавлены:
- Детальное отслеживание попыток подключения
- Разделение таймаутов:
  - Подключение: 10 секунд (было 5)
  - Получение ответа: 5 секунд
- Специализированные исключения для каждого типа ошибки:
  - `TimeoutException` - если сервис не ответил вовремя
  - `IOException` - проблемы с подключением
  - `OperationCanceledException` - операция отменена
  - Общие исключения с деталями
- Состояние подключения: свойство `IsConnected`
- События для отслеживания ошибок: `ErrorOccurred`

#### Логирование включает:
```
- Попытка подключения к [pipeName]
- Статус подключения (успех/ошибка)
- Время отклика (timeout информация)
- Детали ошибок (тип, сообщение)
```

## Как использовать для диагностики

### Способ 1: Встроенное диагностическое окно (Рекомендуется)
1. Запустите оба приложения от администратора
2. Нажмите кнопку "Диагностика" в главном окне
3. Нажмите "Test Connection" или "Start Auto-Test"
4. Смотрите результаты в окне логов

### Способ 2: Просмотр логов
1. Проверьте консоль FireWallService - есть ли "Waiting for GUI connections"
2. Посмотрите firewall.log в папке сервиса
3. Откройте Diagnostics и сохраните логи

### Способ 3: PowerShell команды
```powershell
# Проверить, запущен ли сервис
Get-Process | Where-Object {$_.Name -like "*FireWall*"}

# Просмотреть логи сервиса
Get-Content firewall.log -Tail 50

# Проверить права администратора
[System.Security.Principal.WindowsIdentity]::GetCurrent().Owner
```

## Информация для отладки

### Что проверить если не подключается

1. **Оба приложения от администратора?**
   - Откройте диспетчер задач (Ctrl+Shift+Esc)
   - Посмотрите "User" в столбце процессов

2. **Сервис запущен?**
   ```powershell
   Get-Process FireWallService
   ```

3. **Нет ошибок в консоли сервиса?**
   - Посмотрите консольный вывод сервиса
   - Там должно быть "Waiting for GUI connections"

4. **Какая конкретная ошибка в Diagnostics Window?**
   - "Connection timeout" → сервис не слушает
   - "Access is denied" → недостаточно прав
   - "Connection reset" → сервис упал
   - "Empty response" → сервис не отвечает правильно

5. **Firewall не блокирует?**
   ```powershell
   Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled $False
   ```

## Известные проблемы и их решения

| Проблема | Решение |
|----------|---------|
| Timeout при подключении | Убедитесь, что FireWallService запущен от администратора |
| Access denied | Запустите оба приложения от администратора |
| Empty response | Проверьте firewall.log на ошибки в HandleIpcRequest |
| Connection reset | Перезагрузите сервис и приложение |
| Нет сообщения "Waiting" в консоли | Сервис не дошел до инициализации Named Pipes |
