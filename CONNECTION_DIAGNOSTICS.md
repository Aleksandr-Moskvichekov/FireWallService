# Диагностика проблемы подключения IPC

## Проблема
Приложение FirewallWiewApp не может подключиться к сервису FireWallService через Named Pipes.

## Причины и решения

### 1. **Сервис не запущен**
- Убедитесь, что `FireWallService` запущен как служба Windows или в режиме консоли
- Откройте Services.msc и проверьте статус сервиса
- Или запустите FireWallService.exe вручную в консоли

### 2. **Права доступа**
- **ВАЖНО**: FireWallService должен работать с правами администратора
  - Щелкните правой кнопкой на FireWallService.exe → "Запустить от имени администратора"
  - Приложение FirewallWiewApp также должно иметь права администратора

### 3. **Имя именованного канала**
Оба приложения должны использовать одинаковое имя канала:
- FireWallService: `FireWallServicePipe` (в `NamedPipeServer`)
- FirewallWiewApp: `FireWallServicePipe` (в `IpcClient`)

### 4. **Firewall Windows**
Если локальный файервол Windows включен, он может блокировать именованные каналы:
- Отключите файервол временно для тестирования
- Или добавьте исключение для обоих приложений

## Диагностика

### Способ 1: Встроенное диагностическое окно
1. Запустите приложение FirewallWiewApp
2. Нажмите кнопку "Диагностика" в правом верхнем углу
3. Нажмите "Test Connection"
4. Проверьте логи для деталей ошибки

### Способ 2: Просмотр логов
- **FireWallService**: `firewall.log` в той же папке, где запущен сервис
- **FirewallWiewApp**: 
  - Откройте Diagnostics Window
  - Все ошибки будут видны в окне логов

## Шаги диагностики

### Шаг 1: Убедитесь, что сервис запущен
```powershell
# В PowerShell от имени администратора
Get-Process | Where-Object {$_.Name -like "*FireWall*"}
```

### Шаг 2: Проверьте права доступа
- Оба приложения должны быть запущены от администратора
- Используйте "Запустить от имени администратора"

### Шаг 3: Отключите файервол Windows (временно для тестирования)
```powershell
# Отключить
Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled $False

# Включить обратно
Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled $True
```

### Шаг 4: Используйте диагностическое окно
- Нажмите "Start Auto-Test" для непрерывного тестирования подключения
- Сообщения об ошибках покажут точную проблему

## Типичные ошибки и решения

### "Connection timeout to FireWallServicePipe"
- **Причина**: Сервис не запущен или не слушает канал
- **Решение**: Запустите FireWallService с правами администратора

### "Connection error: Access is denied"
- **Причина**: Недостаточно прав доступа
- **Решение**: Запустите оба приложения от администратора

### "Empty response from service"
- **Причина**: Сервис не ответил корректно
- **Решение**: 
  1. Проверьте firewall.log сервиса
  2. Перезагрузите сервис
  3. Проверьте версии .NET на обоих приложениях

### "Connection reset by peer"
- **Причина**: Сервис закрыл соединение
- **Решение**: Проверьте логи FireWallService

## Рекомендуемый процесс запуска

1. **Откройте PowerShell от имени администратора**
2. **Перейдите в папку с FireWallService.exe**
3. **Запустите сервис:**
   ```powershell
   .\FireWallService.exe
   ```
4. **В другом окне PowerShell откройте приложение:**
   ```powershell
   .\FirewallWiewApp.exe
   ```
5. **Нажмите "Диагностика" и протестируйте подключение**

## Дополнительная информация

### Логирование
IpcClient теперь логирует:
- Попытку подключения
- Таймауты соединения
- Ошибки доступа
- Проблемы с десериализацией JSON
- Времена отклика

### Таймауты
- Подключение: 10 секунд
- Получение ответа: 5 секунд

Если эти значения недостаточны, отредактируйте `IpcClient.cs` и увеличьте таймауты.

## Дальнейшие шаги

Если диагностика не помогла:
1. Проверьте firewall.log для деталей на стороне сервиса
2. Добавьте дополнительное логирование в HandleIpcRequest
3. Используйте netstat для проверки открытых каналов:
   ```powershell
   Get-NetTCPConnection | Where-Object {$_.LocalPort -like "*pipe*"}
   ```
