# FireWallService - Межсетевой экран для Windows

## 📋 Описание

Полнофункциональная система управления межсетевым экраном для Windows, состоящая из:

- **FireWallService** - фоновый сервис для перехвата и фильтрации сетевых пакетов
- **FirewallWiewApp** - графический интерфейс (GUI) для управления сервисом

Система использует именованные каналы (Named Pipes) для взаимодействия между компонентами.

## 🚀 Быстрый старт

### Требования
- Windows 7 или выше
- .NET 10.0 Runtime установлен
- **Права администратора** на обоих приложениях
- WinDivert драйверы установлены

### Запуск

1. **Откройте PowerShell от имени администратора:**
```powershell
Start-Process powershell -Verb RunAs
```

2. **Запустите FireWallService:**
```powershell
cd "D:\Документы\MVS Programs\FireWallService\FireWallService\bin\Debug\net10.0"
.\FireWallService.exe
```

Вы должны увидеть:
```
╔════════════════════════════════════════════════════════╗
║   FireWallService Started                              ║
║   Named Pipe: FireWallServicePipe                      ║
║   Waiting for GUI connections...                       ║
╚════════════════════════════════════════════════════════╝
```

3. **В другом PowerShell окне (также от администратора) запустите приложение:**
```powershell
cd "D:\Документы\MVS Programs\FireWallService\FirewallWiewApp\bin\Debug\net10.0-windows"
.\FirewallWiewApp.exe
```

4. **Проверьте подключение:**
   - Нажмите кнопку "Диагностика" в правом верхнем углу
   - Нажмите "Test Connection"
   - Должно быть сообщение об успешном подключении

## 🐛 Диагностика проблем

### Проблема: "Connection timeout"
**Решение:** Убедитесь, что FireWallService запущен от администратора

### Проблема: "Access is denied"
**Решение:** Запустите оба приложения от администратора

### Проблема: "Сервис не запускается"
**Решение:** Проверьте файл `firewall.log` в папке сервиса

Для подробного решения проблем см. [TROUBLESHOOTING.md](TROUBLESHOOTING.md)

## 🔍 Дополнительная информация

### Команды консоли сервиса

При запуске в режиме консоли используйте команды:

```
help              - показать справку
status            - статус сервиса
rules             - список всех правил
packets           - последние пакеты
stats             - статистика
addrule           - добавить правило
removerule        - удалить правило
clear-packets     - очистить историю пакетов
clear-logs        - очистить логи
start             - начать захват
stop              - остановить захват
exit              - выход
```

### Файлы конфигурации

- `firewall.log` - логи сервиса (уровень Debug)
- `appsettings.json` - конфигурация приложения
- `captures/firewall_capture.pcap` - запись захватанных пакетов

### Порты и каналы

- **Named Pipe:** `FireWallServicePipe` (локальный)
- **Таймауты:**
  - Подключение: 10 секунд
  - Ответ: 5 секунд
